## Why

`pnpm run verify:ship:local` can run targeted UI smoke followed by the full UI
smoke suite in one command. Targeted auth smoke may consume the one-time
allowlisted signup email generated by `smoke:bootstrap`, which leaves the later
full suite with a stale bootstrap file and causes duplicate-signup failures.

## What Changes

- Refresh UI smoke bootstrap data before the full suite starts when the runner
  executes targeted smoke and then full smoke in the same invocation.
- Preserve the existing Supabase stack and app server; only reseed the smoke
  fixtures and regenerate the bootstrap file between those suites.
- Keep targeted-only and full-only smoke behavior unchanged.

## Capabilities

### Modified Capabilities
- `openspec-automation`: Release verification smoke runner behavior.

## Impact

- Affects `scripts/run-ui-smoke.ts` and its unit coverage.
- Verification should include direct script tests and the release smoke gate that
  originally exposed the stale bootstrap failure.
